Overview

Faster web deliver delightful user expedience. statico was created to make the complicated optimization processes simple and easy to speed up web. It's a tool developed by developer for developer. Hope you enjoy!

Hello World

A simple web page (source code on github) contain CSS, images and JavaScripts demonstrate how optimization works in action:

{% highlight text %} wwwroot ├── index.html └── assets ├── css │ └── homepage.css ├── images │ └── logo.png ├── js │ └── homepage.js └── vendors └── jquery-1.11.0 └── jquery-1.11.0.js {% endhighlight %}

Next tell the statico cloud service what to optimize in config.gradle:

{% highlight groovy %} statico { inputWwwRoots = ['wwwroot/'] outputWwwRoot = 'build/optimized-files/' inputFilePathPatterns = [ '/index.html' ] } {% endhighlight %}

Then you will get the optimized index.html to store in your server. All referenced static assets will be stored in S3 and delivered by CloudFront:

{% highlight html %} hello world - statico.io

Hello statico

My favorite color is blue
{% endhighlight %}

The result from webpagetest.org speed index: original 701 vs 300 optimized

Best Practices

statico is a lossless optimization solution, we made it easy to apply the optimization best practices. There are well known high performance web sites best practices which improve render performance. The most commonly ignored rules is Move scripts to the Bottom of the page. For web site, I strongly suggest to avoid rely on JavaScript for rendering main content.

There are ideas which you can play to get better rendering performance:

  • Critical Rendering Path by Patrick Sexton
  • CSS and the critical path by Patrick Hamann
  • Analyzing Critical Rendering Path Performance from Google
