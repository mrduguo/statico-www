The challenge

It is complicated and require heavy engineering effort to make the web fast.

As a web designer or front end developer, you may well know what need to be done and open source tools can be used to make it fast. In reality, there are very few websites able to achieve best front end performance. Even for big brands such as ebay with professional teams, they still perform poor on front end due to the complicity involved with optimization and integration of development process.

Given listing site landing page comparison as an example, WhatClinic.com user will see in 0.5 second while ebay.com user have to wait for 3.5 seconds in webpagetest.org tests with standard cable network profile.

Our solution: front end optimization as a service

To solve the problem to make the web faster, we created statico.io as a cloud service to make front end optimization easy and simple as 1, 2, 3:

1. Annotate your static assets e.g.
  • inline css
    • critical.css?sio-css-embed=enabled
  • remove unused css
    • clean.css?sio-css-uncss=enabled
  • progressive render web fonts
    • cool-fonts.css?sio-progressive-load=enabled
  • inline image
    • url(logo.png?sio-data-url=enabled)
2. Optimize with SDK automatically
  1. scan your source code
  2. upload all referenced files
  3. optimize based on annotation
  4. give back optimized file with external referenced resource delivered by CDN
  5. store the optimized file to your server
3. Enjoy
  • Speedy!
